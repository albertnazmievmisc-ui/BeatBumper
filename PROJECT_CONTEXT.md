Бомбы парсятся, но не отображаются визуально и не наносят штраф 2. Нет поддержки стен (obstacles) 3. Нет поддержки арок (arcs) - слайдеры 4. Переменный BPM не учитывается 5. Требуется ручная калибровка audio_latency_ms 6. Рекорды сохраняются, но нет отображения лучшего результата в меню ### 🔧 КРИТИЧЕСКИЕ ИСПРАВЛЕНИЯ (История багов) #### Баг #1: Note() takes no arguments **Причина:** В классе Note не было метода __init__ или аннотаций dataclass **Решение:** Использовать @dataclass с полными аннотациями типов **Урок:** ВСЕГДА проверять, что классы с данными имеют либо __init__, либо @dataclass #### Баг #2: Circular import (cannot import HitResult) **Причина:** В note_controller.py был импорт from core.note_controller import HitResult **Решение:** Определять Enum в том же файле, не импортировать из себя **Урок:** НИКОГДА не импортировать из модуля самого себя. Перечисления нужно определять в том же файле, если они используются только там #### Баг #3: IndentationError в score_manager.py **Причина:** Неправильные отступы после добавления методов save_high_score **Решение:** Полная перезапись файла с правильной структурой **Урок:** При добавлении методов в существующие классы всегда проверять отступы. Проще перезаписать файл целиком #### Баг #4: ImportError при запуске из-за циклических зависимостей **Причина:** game_screen импортирует note_controller, который импортирует config, который может импортировать что-то ещё **Решение:** Все импорты должны быть на уровне модуля, не создавать круговых зависимостей **Урок:** Держать импорты плоскими. Утилиты не должны импортировать core, core может импортировать утилиты #### Баг #5: scores.json не создавался **Причина:** В game_screen._end_game не было вызова save_high_score **Решение:** Добавить вызов метода с правильными параметрами **Урок:** Всегда проверять, что feature полностью интегрирован, а не только добавлен класс --- ## 🏗️ Архитектура (исправленная) BeatBumper/ ├── main.py # Точка входа ├── src/ │ ├── core/ │ │ ├── game_engine.py # Игровой цикл, состояния │ │ ├── note_controller.py # ВАЖНО: HitResult определен здесь же │ │ └── score_manager.py # Полная поддержка рекордов │ ├── parser/ │ │ ├── beatmap_loader.py # Поддержка v2/v3/v4 Info.dat │ │ └── beatmap_parser.py # Парсинг v2/v3/v4 + бомбы │ ├── audio/ │ │ └── audio_manager.py # + компенсация задержки │ ├── render/ │ │ ├── renderer.py │ │ ├── note_sprite.py │ │ └── effects.py # + create_miss_effect() │ ├── input/ │ │ └── input_handler.py # + поддержка D-pad (JOYHATMOTION) │ ├── ui/ │ │ ├── menu_screen.py │ │ ├── game_screen.py # + song_name, song_difficulty │ │ └── results_screen.py │ └── utils/ │ ├── config.py │ └── helpers.py ├── assets/ │ └── sounds/ # hit.wav, miss.wav (создаются автоматически) ├── songs/ # Beat Saber .zip мапы ├── scores.json # АВТОМАТИЧЕСКИ СОЗДАЕТСЯ при игре ├── config.json # Настройки └── PROJECT_CONTEXT.md # Этот файл --- ## ⚠️ ВАЖНЫЕ ПРАВИЛА ДЛЯ ИЗБЕЖАНИЯ БАГОВ ### 1. Определение Enum ПРАВИЛЬНО: ```python class HitResult(Enum): PERFECT = auto() GOOD = auto() ``` НЕПРАВИЛЬНО: ```python from .note_controller import HitResult # ЦИКЛИЧЕСКИЙ ИМПОРТ! ``` ### 2. Классы с данными ПРАВИЛЬНО: ```python @dataclass class Note: time: float color: NoteColor hit: bool = False ``` НЕПРАВИЛЬНО: ```python class Note: pass # Без __init__ или dataclass ``` ### 3. Отступы при добавлении методов ПРАВИЛЬНО: перезаписать весь файл НЕПРАВИЛЬНО: пытаться вставить метод с правильными отступами в середину ### 4. Интеграция фич ПРАВИЛЬНО: проверить, что метод ВЫЗЫВАЕТСЯ, а не только определен НЕПРАВИЛЬНО: добавить метод save_high_score, но не вызвать его ### 5. Импорты ПРАВИЛЬНО: ```python from utils.config import Config from core.note_controller import NoteController ``` НЕПРАВИЛЬНО: ```python from src.core.note_controller import NoteController # лишний src ``` --- ## 🎮 Управление (ПОЛНОЕ) | Действие | Клавиатура | Steam Deck | |----------|-----------|------------| | Удар красной | → / D | RT | | Удар синей | ← / A | LT | | Пауза | ESC / P | Start | | Выход из игры | Q | Back/Select | | Меню вверх | ↑ | D-pad вверх | | Меню вниз | ↓ | D-pad вниз | | Выбор | ENTER | A | | Назад | ESC | B | | Полный экран | Alt+Enter | - | --- ## 📝 Система рекордов (scores.json) Файл создается АВТОМАТИЧЕСКИ в корне проекта после завершения песни. ЧЕК-ЛИСТ 1. **Note() takes no arguments** → Проверить класс Note в note_controller.py → Должен быть @dataclass с аннотациями 2. **Circular import** → Проверить, нет ли импорта из самого себя → HitResult должен быть определен в том же файле 3. **IndentationError** → Полностью переписать файл, не редактировать частично 4. **Нет scores.json после игры** → Проверить, что в game_screen._end_game вызывается save_high_score → Добавить print(f"DEBUG: {self.song_name}") для отладки 5. **Не работает D-pad** → Проверить, что в input_handler.py есть обработка JOYHATMOTION 6. **Нет звука** → Проверить assets/sounds/hit.wav и miss.wav → Скрипт создаст заглушки автоматически 7. **Ноты не синхронизированы** → Настроить audio_latency_ms в config.json → Обычно от -100 до 100, для Steam Deck: -50 --- ## 🚀 Быстрый старт после клонирования ```bash cd BeatBumper python3 -m venv .venv source .venv/bin/activate pip install -r requirements.txt python3 main.py ``` Если ошибки -
